

# خطة تحويل HN-STOCK من Replit إلى Lovable

## الوضع الحالي
HN-STOCK يعمل على Replit بتقنيات **React 19 + Express 5 + PostgreSQL** ويوفر نظام تخزين وتوزيع (Fulfillment) مع 5 أدوار و3 لغات.

## التحدي
Lovable لا يدعم سيرفرات خلفية (Express/Node.js). لذلك يجب تحويل البنية:

| المكون | Replit (حالياً) | Lovable (بعد التحويل) |
|--------|----------------|----------------------|
| الواجهة | React 19 | React 18 (Lovable) |
| الـ API | Express 5 | Edge Functions |
| قاعدة البيانات | PostgreSQL محلي | Lovable Cloud DB |
| الاستضافة | Replit | Lovable + السيرفر الخاص |

## خطوات التنفيذ

### 1. إنشاء جداول قاعدة البيانات
- نقل جداول HN-STOCK (المنتجات، الطلبات، الشحنات، المستودعات، المستخدمين) إلى Lovable Cloud
- إضافة سياسات RLS للحماية

### 2. تحويل Express API إلى Edge Functions
- كل endpoint في Express يصبح Edge Function مستقلة
- مثال: `/api/orders` → `hn-stock-orders/index.ts`
- `/api/shipments` → `hn-stock-shipments/index.ts`

### 3. بناء واجهات HN-STOCK
- لوحة تحكم المخزون
- إدارة الطلبات والشحنات (8 حالات تشغيلية)
- مركز الاتصال لتأكيد الطلبات
- تتبع الشحنات + دعم COD
- دعم 3 لغات (عربي، فرنسي، إنجليزي)

### 4. ربط HN-STOCK ببوابة HN Groupe
- إضافة رابط في صفحة `/hn-groupe` للوصول للمنصة

## ما أحتاجه منك

لتنفيذ هذا، أحتاج:
1. **كود المشروع من Replit** — شارك رابط المشروع أو ارفع الملفات الأساسية (خاصة ملفات API والـ schema)
2. **هيكل قاعدة البيانات** — ملف SQL أو schema للجداول الموجودة

## ملاحظة مهمة
المهام الخلفية المتقدمة (بوتات، كشط ويب مجدول) التي كانت على Replit يمكن إبقاؤها على Replit ومزامنتها عبر GitHub كما هو معتمد حالياً.
